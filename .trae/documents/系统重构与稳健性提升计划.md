## 1. 重构 PlaygroundService (消除技术债务)
目前 `PlaygroundService` 中的 `runPrompt` 方法过于复杂，且包含大量硬编码配置。为了支持后续的参数配置功能，我们需要先进行重构。

### A. 配置外置化
- 将硬编码的模型名称、API URL 和价格信息提取到 `application.yml` 中。
- 创建 `GenieConfig` 类来映射这些配置，方便统一管理。

### B. 引入策略模式 (Strategy Pattern)
- 定义 `GenerationStrategy` 接口。
- 拆分实现以下策略类：
    - `TextGenerationStrategy`: 处理 Qwen 等文本模型。
    - `ImageGenerationStrategy`: 处理 Wanx-V1 等图片模型。
    - `VideoGenerationStrategy`: 处理 Wanx 2.1/2.6 等视频模型（包含异步轮询逻辑）。
- `PlaygroundService` 将充当 Context，根据请求类型分发给相应的策略。

## 2. 增强前端错误处理
目前前端 API 调用出错时反馈较弱，容易让用户困惑。

- **全局提示组件**: 引入或封装 Toast/Notification 组件。
- **拦截器增强**: 在 `api.ts` 的 Axios 拦截器中集成全局错误提示，当 API 返回 4xx/5xx 错误时自动弹出友好的提示信息。

## 3. 下一步功能准备 (Phase 1: 参数控制)
在完成上述重构后，代码结构将更清晰，我们将能够更容易地实现 roadmap 中的“参数精细化控制”功能（如 Temperature, Seed, 视频时长等），因为这些参数可以直接透传给相应的策略类处理。
