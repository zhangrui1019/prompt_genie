# 前后端代码规范

## 1. 后端代码规范

### 1.1 编码风格
- **缩进**：使用 4 个空格进行缩进
- **换行**：每行不超过 120 个字符
- **空格**：运算符两侧、逗号后、大括号前后都要加空格
- **空行**：方法之间、逻辑块之间使用空行分隔
- **注释**：使用 Javadoc 风格的注释，包括类、方法和关键逻辑

### 1.2 命名约定
- **类名**：使用驼峰命名法，首字母大写（如 `UserService`）
- **方法名**：使用驼峰命名法，首字母小写（如 `getUserById`）
- **变量名**：使用驼峰命名法，首字母小写（如 `userId`）
- **常量**：使用全大写，下划线分隔（如 `MAX_PAGE_SIZE`）
- **包名**：使用小写字母，点分隔（如 `com.promptgenie.service`）

### 1.3 目录结构
```
backend/
├── src/
│   ├── main/
│   │   ├── java/
│   │   │   └── com/
│   │   │       └── promptgenie/
│   │   │           ├── api/            # API 控制器
│   │   │           ├── core/            # 核心功能
│   │   │           ├── dto/             # 数据传输对象
│   │   │           ├── entity/          # 实体类
│   │   │           ├── mapper/          # MyBatis Mapper
│   │   │           ├── service/         # 业务逻辑
│   │   │           └── utils/          # 工具类
│   │   └── resources/                  # 资源文件
│   └── test/                           # 测试代码
└── pom.xml                             # Maven 配置
```

### 1.4 代码组织
- **分层架构**：严格按照控制器、服务、数据访问层分层
- **单一职责**：每个类只负责一个功能
- **依赖注入**：使用 Spring 的依赖注入机制
- **异常处理**：统一的异常处理机制
- **日志记录**：使用 SLF4J 进行日志记录

### 1.5 数据库操作
- **ORM**：使用 MyBatis-Plus 进行数据库操作
- **事务管理**：使用 Spring 的事务管理
- **SQL 规范**：避免复杂的 SQL，使用参数化查询
- **连接池**：使用 HikariCP 连接池

### 1.6 API 设计
- **RESTful**：遵循 RESTful API 设计规范
- **路径**：使用小写字母，单词之间用连字符分隔
- **方法**：使用 HTTP 方法（GET、POST、PUT、DELETE）
- **参数**：使用 @RequestParam、@PathVariable、@RequestBody
- **响应**：统一的响应格式，包含状态码、消息和数据

## 2. 前端代码规范

### 2.1 编码风格
- **缩进**：使用 2 个空格进行缩进
- **换行**：每行不超过 100 个字符
- **空格**：运算符两侧、逗号后、大括号前后都要加空格
- **空行**：组件之间、逻辑块之间使用空行分隔
- **注释**：使用 JSDoc 风格的注释，包括组件、函数和关键逻辑

### 2.2 命名约定
- **组件名**：使用 PascalCase（如 `UserList`）
- **函数名**：使用 camelCase（如 `getUserData`）
- **变量名**：使用 camelCase（如 `userName`）
- **常量**：使用全大写，下划线分隔（如 `MAX_PAGE_SIZE`）
- **文件命名**：组件文件使用 PascalCase，其他文件使用 camelCase

### 2.3 目录结构
```
frontend/
├── public/             # 静态资源
├── src/
│   ├── assets/         # 资源文件
│   ├── components/     # 通用组件
│   ├── hooks/          # 自定义 hooks
│   ├── lib/            # 工具库
│   ├── pages/          # 页面组件
│   ├── services/       # API 服务
│   ├── store/          # 状态管理
│   ├── types/          # TypeScript 类型定义
│   ├── App.tsx         # 应用入口
│   └── main.tsx        # 主入口
├── index.html          # HTML 模板
├── tsconfig.json       # TypeScript 配置
├── package.json        # 依赖配置
└── vite.config.ts      # Vite 配置
```

### 2.4 代码组织
- **组件设计**：使用函数式组件，优先使用 Hooks
- **状态管理**：使用 Zustand 进行状态管理
- **样式**：使用 TailwindCSS 进行样式管理
- **类型安全**：使用 TypeScript 确保类型安全
- **代码分割**：使用动态导入进行代码分割

### 2.5 API 调用
- **封装**：封装 API 调用，统一处理错误和加载状态
- **请求方法**：使用 axios 进行 HTTP 请求
- **拦截器**：使用请求和响应拦截器
- **错误处理**：统一的错误处理机制
- **取消请求**：支持取消未完成的请求

### 2.6 组件设计
- **Props 类型**：使用 TypeScript 定义 Props 类型
- **默认值**：为 Props 设置默认值
- **事件处理**：使用箭头函数或 useCallback 优化性能
- **状态管理**：使用 useState 和 useReducer 管理组件状态
- **副作用**：使用 useEffect 处理副作用

## 3. 通用规范

### 3.1 版本控制
- **提交信息**：使用清晰的提交信息，格式为 `[类型] 描述`
- **提交粒度**：每个提交只包含一个功能或修复
- **分支管理**：遵循 Git Flow 分支管理策略
- **标签**：使用语义化版本号进行标签管理

### 3.2 依赖管理
- **版本固定**：使用固定版本号，避免版本冲突
- **依赖更新**：定期更新依赖，确保安全性
- **依赖清理**：移除未使用的依赖

### 3.3 性能优化
- **后端**：使用缓存、优化 SQL、合理使用线程池
- **前端**：使用 memo、useMemo、useCallback 优化渲染性能
- **网络**：使用 HTTP/2、压缩、CDN 等优化网络性能

### 3.4 安全性
- **后端**：使用 HTTPS、CSRF 保护、SQL 注入防护
- **前端**：使用 CSP、XSS 防护、输入验证
- **认证**：使用 JWT 进行认证
- **授权**：使用基于角色的访问控制

### 3.5 测试
- **单元测试**：为核心功能编写单元测试
- **集成测试**：测试组件之间的交互
- **端到端测试**：测试完整的业务流程
- **测试覆盖率**：确保测试覆盖率不低于 80%

### 3.6 文档
- **代码文档**：为关键代码编写文档
- **API 文档**：使用 Swagger 生成 API 文档
- **架构文档**：描述系统架构和设计决策
- **部署文档**：描述部署步骤和环境配置
