# Prompt Genie 系统功能文档

## 1. 系统概述

Prompt Genie 是一个智能体与应用构建平台，提供可视化的智能体编排、专业评测和知识库管理功能。系统已实现 V1.3 核心升级，包含智能体画布、专业评测中心和知识库引擎等核心功能。

## 2. 核心功能模块

### 2.1 智能体画布 (Agent Canvas)

**功能描述**：提供可视化的拖拽式画布，用于编排和管理智能体工作流。

**核心功能**：
- **状态机编排**：支持节点与边的概念，实现条件分支、循环和错误重试回路
- **人机协同中断**：在风险节点挂起任务，等待人工审核和纠偏
- **多智能体协作**：每个节点可以作为独立的 Agent，实现专家级协作
- **工具挂载**：无缝集成外部工具，如 Google Search、数据库查询等
- **安全边界设定**：为每个节点设置输出校验规则，确保输出符合预期
- **应用发布**：将构建好的智能体发布为 Web App 或第三方机器人

**技术实现**：
- 后端：Spring Boot 实现的状态机执行引擎 (`AgentExecutorService`)
- 前端：React Flow 实现的可视化画布 (`AgentBuilder.tsx`)
- 支持的节点类型：LLM Node、Tool Node、Human Approval Node、Condition Node、Loop Node、Error Retry Node

### 2.2 专业评测中心 (Evaluation Center)

**功能描述**：提供测试集管理、批量评测和效果对比功能，帮助用户量化智能体和 Prompt 的性能。

**核心功能**：
- **测试集驱动**：支持上传和管理测试用例（输入变量 + 预期输出）
- **批量回归测试**：一键运行多个测试用例，提供通过率报表
- **LLM-as-a-Judge**：使用高阶模型作为裁判，对主观题进行自动打分
- **A/B 效果对比**：对比不同 Prompt 或智能体在同一测试集上的表现

**技术实现**：
- 后端：`EvaluationService` 实现批量评测逻辑
- 前端：`EvaluationCreate.tsx`、`EvaluationList.tsx`、`EvaluationReport.tsx` 实现评测界面

### 2.3 知识库引擎 (RAG Engine)

**功能描述**：提供文档解析、知识存储和检索功能，支持智能体与知识库的集成。

**核心功能**：
- **文档解析与切分**：支持多格式文档解析与自动分块
- **混合检索**：支持向量检索与全文检索结合
- **知识挂载**：在智能体中挂载知识库，实现“所问即所答”

**技术实现**：
- 后端：`KnowledgeService` 实现知识库管理
- 前端：`KnowledgeBase.tsx` 实现知识库管理界面

### 2.4 智能路由网关 (Smart Router)

**功能描述**：根据任务类型和用户配置，智能路由请求到最合适的模型。

**核心功能**：
- **模型竞技场**：自动将流量分发给多个模型，实时对比效果
- **成本/效果路由**：根据用户配置自动选择模型
- **故障自动切换**：主模型故障时自动切换到备用模型

**技术实现**：
- 后端：`SmartRouterService` 实现路由逻辑

### 2.5 自动优化闭环 (Auto-Optimizer Loop)

**功能描述**：基于用户反馈持续优化 Prompt 和智能体性能。

**核心功能**：
- **反馈收集**：API 调用方对结果进行评价
- **自动迭代**：定期收集负面样本，自动生成新版 Prompt 并进行 A/B 测试

**技术实现**：
- 后端：`AutoOptimizerService` 实现自动优化逻辑

## 3. 技术架构

### 3.1 技术栈

- **前端**：React 18, TypeScript, TailwindCSS, Lucide Icons, Vite, Zustand
- **后端**：Java 17, Spring Boot 3.2.2, MyBatis-Plus, PostgreSQL
- **大模型**：阿里云 DashScope (Qwen 模型)
- **可视化**：React Flow

### 3.2 核心模块架构

#### 智能体执行引擎
- `AgentState`：定义智能体执行状态
- `AgentExecutorService`：实现状态机执行逻辑
- `AgentController`：提供智能体运行和恢复的 API

#### 评测中心
- `EvaluationService`：实现批量评测和 LLM-as-a-Judge 功能
- `EvaluationController`：提供评测任务管理的 API
- 前端评测界面：支持测试集管理、评测任务创建和结果查看

#### 知识库引擎
- `KnowledgeService`：实现文档解析、切分和检索功能
- `KnowledgeController`：提供知识库管理的 API
- 前端知识库界面：支持文档上传和知识库管理

## 4. API 接口

### 4.1 智能体相关
- `POST /api/agents/{id}/run`：启动智能体执行
- `POST /api/agents/{id}/resume`：恢复挂起的智能体执行
- `GET /api/agents`：获取智能体列表
- `POST /api/agents`：创建智能体
- `PUT /api/agents/{id}`：更新智能体
- `DELETE /api/agents/{id}`：删除智能体

### 4.2 评测相关
- `POST /api/evaluations`：创建评测任务
- `GET /api/evaluations`：获取评测任务列表
- `GET /api/evaluations/{id}/results`：获取评测结果
- `DELETE /api/evaluations/{id}`：删除评测任务

### 4.3 知识库相关
- `POST /api/knowledge/upload`：上传文档
- `GET /api/knowledge/bases`：获取知识库列表
- `POST /api/knowledge/bases`：创建知识库
- `PUT /api/knowledge/bases/{id}`：更新知识库
- `DELETE /api/knowledge/bases/{id}`：删除知识库

## 5. 系统使用流程

### 5.1 智能体构建流程
1. **创建智能体**：在 AgentBuilder 页面创建新的智能体
2. **添加节点**：从工具栏添加不同类型的节点
3. **配置节点**：设置节点的参数和属性
4. **连接节点**：建立节点之间的逻辑关系
5. **保存智能体**：将智能体配置保存到系统
6. **运行智能体**：启动智能体执行流程
7. **人工审核**：在需要时对挂起的任务进行审核

### 5.2 评测流程
1. **创建测试集**：上传或手动添加测试用例
2. **创建评测任务**：选择要评测的智能体或 Prompt
3. **运行评测**：系统自动执行测试用例
4. **查看结果**：查看评测报告和详细结果
5. **A/B 对比**：对比不同版本的性能

### 5.3 知识库使用流程
1. **创建知识库**：在 KnowledgeBase 页面创建新的知识库
2. **上传文档**：支持多种格式的文档上传
3. **管理知识库**：查看和管理已上传的文档
4. **挂载知识库**：在智能体中挂载知识库

Prompt Genie 系统已完成 V1.3 核心升级，实现了智能体画布、专业评测中心和知识库引擎等核心功能。系统采用现代化的技术栈，提供了直观的用户界面和强大的功能，为用户提供了一个完整的智能体构建和管理平台。

未来，系统将继续演进，不断增强功能和性能，为用户提供更加全面和强大的智能体构建平台。
