# 产品规则

## 1. 设计原则
- **用户中心**：优先考虑用户体验和需求
- **简洁易用**：界面清晰，操作流程简单直观
- **功能实用**：提供用户真正需要的功能，避免过度设计
- **性能优先**：确保系统响应迅速，操作流畅
- **安全可靠**：保障数据安全，系统稳定运行
- **一致性**：保持界面风格和操作方式的一致性
- **可扩展性**：设计具有可扩展性的系统架构

## 2. 核心功能
- **智能体画布**：可视化拖拽式界面，支持状态机编排、人机协同、多智能体协作、工具集成、安全边界设定和应用发布
- **评测中心**：测试集管理、批量回归测试、LLM-as-a-Judge、A/B 效果对比、详细评测报告
- **知识库引擎**：文档解析、智能切分、向量嵌入、混合检索、知识挂载
- **智能路由网关**：模型竞技场、成本/效果路由、故障自动切换
- **自动优化闭环**：反馈收集、自动迭代

## 3. 用户体验
- **界面设计**：清晰布局，统一色彩，易读字体，直观图标，响应式布局
- **交互设计**：简单流程，及时反馈，友好错误提示，必要引导，个性化设置
- **性能要求**：页面加载 < 3 秒，操作响应 < 1 秒，系统稳定，支持大量用户

## 4. 安全规范
- **数据安全**：传输和存储加密，严格访问控制，定期备份，敏感数据脱敏
- **系统安全**：安全身份认证，基于角色的授权，定期漏洞扫描，完整操作日志
- **代码安全**：定期代码审查，及时更新依赖，进行安全测试

## 5. 性能要求
- **系统性能**：API 响应 < 500ms，支持 1000+ 并发用户，合理资源利用
- **数据库性能**：优化查询，合理索引，高效事务处理，配置连接池
- **前端性能**：优化页面加载，减少资源大小，提高渲染速度，合理缓存策略

## 6. 部署与运维
- **环境要求**：开发、测试、生产环境分离，配置合理
- **部署流程**：自动化部署，严格版本管理，完善回滚机制，监控告警
- **运维规范**：实时监控，集中日志管理，容量规划，灾备方案

## 7. 迭代与质量
- **迭代流程**：需求收集 → 分析 → 设计 → 开发 → 测试 → 发布 → 评估
- **版本管理**：语义化版本号，合理发布计划，版本兼容，详细升级指南
- **质量保证**：单元测试、集成测试、端到端测试、性能测试、安全测试
- **质量指标**：功能覆盖率 100%，测试通过率 ≥ 95%，控制缺陷率，定期收集用户反馈

## 8. 文档规范
- **文档类型**：需求文档、设计文档、API 文档、用户文档、部署文档
- **文档标准**：内容完整，准确一致，结构清晰，及时更新

## 9. 代码注释规范
- **类注释**：每个类都必须有类注释，说明类的功能、用途和主要职责
- **方法注释**：每个方法都必须有方法注释，说明方法的功能、参数、返回值和异常
- **变量注释**：对于复杂的变量或常量，必须添加注释说明其用途和含义
- **代码块注释**：对于复杂的代码块，必须添加注释说明其逻辑和实现思路
- **注释格式**：使用标准的 Javadoc 格式（Java）或 JSDoc 格式（TypeScript/JavaScript）
- **注释语言**：使用英文编写注释，确保团队成员都能理解
- **注释更新**：当代码发生变更时，必须同步更新相关的注释
